Week 7 Homework

In this exercise, you will pick up where the last homework left off – your application will reach out to TMDB to get a list of movies and allow a user to create maintain which movies have been watched – with the addition of user login and a per-user "watch list".

A possible solution for last week's homework has been provided as a starting point. Note: it's the basic solution without the ability to "un-watch" a movie. Your goal is to take it from there, adding user registration/sign-in and using the currently signed-in user to determine which movies have been watched.

Things to Know

  • The movies.html file includes all the necessary scripts and links for talking to the Firebase API; the only thing missing is your app-specific Firebase configuration – copy that from your personal Firebase account.
  • The code to call the TMDB API is provided, but you'll need to provide your own TMDB API key in movies.js.
  • Do your work in movies.js, but don't be afraid to start over when you get stuck! Starting over is often the best way to clear your mind. You can refer to movies-start.js if you want to start from scratch.

Requirements and Rubric

(These are high-level requirements for grading; more detailed mechanics are included in the comments of movies.js)

  • Create a new brand-new repository in your GitHub account called hw7. Clone and open the code in VSCode. Add an HTML page and corresponding JS file – you can use the provided movies.html and movies.js as a starting point.
  • (2 points) Implement a login flow – when the user is signed-out, display a sign-in interface.
  • (4 points) When the user is signed-in, display the user's name and a link to sign-out. Ensure that a users collection is present in your database and a document is present for each user who signs-in to your application.
  • (4 points) Ensure the "watch list" is user-specific, i.e. a separate one is maintained for each user who signs-in to your application.
